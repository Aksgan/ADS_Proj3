Data corruption issues identified |accessdate=2007-12-27 |publisher=Microsoft |date=2007-10-02 |archiveurl = http://web.archive.org/web/20071231144315/http://forums.microsoft.com/WindowsHomeServer/ShowPost.aspx?PostID=2234532&SiteID=50 <!-- Bot retrieved archive --> |archivedate = 2007-12-31}}</ref> [[Microsoft]] formally warned users of the seriousness of the flaw on 20 December 2007. Microsoft then issued a list of applications, including [[Windows Live Photo Gallery]], [[Microsoft OneNote]], [[Microsoft Outlook]] and [[SyncToy]] 2.0, which might have triggered the flaw if they were used to edit the files on a WHS share directly.<ref name="MS-support-1" /> This issue was fixed by Power Pack 1, released on July 21, 2008.<ref name="powerpack1"/> === No native backup === Windows Home Server RTM did not include a mechanism for backing up the server. Power Pack 1 added the ability to back up files stored on the Shared Folders, to an external drive.<ref name="powerpack1" /> Users can also subscribe to 3rd-party online services, for a fee. However, there remains no way to back up the installed server operating system. Backing-up of the client backup database is available either manually using the instructions provided by Microsoft on page 24 of [http://download.microsoft.com/download/1/8/0/18096c95-4850-4176-9821-970691b98aaf/Windows_Home_Server_Technical_Brief_-_Home_Computer_Backup_and_Restore.docx this document] or can be done using the WHS BDBB add-in written by Alex Kuretz and available from [http://www.mediasmartserver.net/add-ins/ this website]. ==Pricing== While some hardware manufacturers have developed dedicated boxes,<ref>{{ cite web |title=List of dedicated WHS boxes offered by hardware manufacturers |url=http://www.microsoft.com/windows/products/winfamily/windowshomeserver/buy.mspx}}</ref> Microsoft has also released Windows Home Server under the [[Original equipment manufacturer|OEM]]/System Builder license.
